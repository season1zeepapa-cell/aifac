# Brief: DocStore 크롤링 & 지식화 시스템

> 네이버 뉴스 API + 사이트 게시판 크롤링으로 외부 콘텐츠를 검색·점수화·선택적으로 지식 DB에 저장하는 시스템

## Requirements
- [ ] 1. **네이버 뉴스 검색**: 키워드 입력 → 네이버 검색 API 호출 → 결과 목록 미리보기 (키워드당 최대 건수 UI 설정, 기본 20건, 최대 100건)
- [ ] 2. **사이트 게시판 크롤링**: 등록된 사이트(KISA, 개인정보포털, 개인정보보호위원회)의 공지사항·보도자료 게시판에서 최근 N일(기본 7일) 내 게시물 수집
- [ ] 3. **크롤링 소스 관리 UI**: 사이트 URL 추가/삭제/활성화, 게시판 경로 설정
- [ ] 4. **키워드 관리 UI**: 검색 키워드 추가/삭제, 키워드별 최대 수집 건수 설정
- [ ] 5. **점수 가중치 UI**: 제목 매칭 가중치, 내용 매칭 가중치를 숫자 입력으로 조절
- [ ] 6. **제외 사이트 관리**: URL 패턴 기반 블랙리스트 추가/삭제
- [ ] 7. **결과 미리보기**: 크롤링 결과를 제목·점수·출처 목록으로 표시, 체크박스 선택
- [ ] 8. **선택적 지식화**: 선택한 항목을 documents 테이블에 저장 → 섹션 분리 + 임베딩 자동 처리
- [ ] 9. **중복 체크**: 동일 URL이 이미 DB에 있으면 스킵
- [ ] 10. **크롤링 범위 설정**: "최근 N일" UI에서 변경 가능
- [ ] 11. **DB 테이블 추가**: `crawl_sources`(사이트 정보), `crawl_keywords`(키워드+설정), `crawl_results`(수집 결과 임시 저장)

## Constraints
- 네이버 API: 환경변수(`NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`)로 관리
- 기존 docstore SPA 구조 유지 (index.html + api/*.js 서버리스)
- Vercel 서버리스 함수 실행 시간 제한 고려 (maxDuration 설정)
- 크롤링 대상 사이트의 robots.txt 준수
- 1차 구현은 수동 실행만 (스케줄링 미구현)

## Non-goals
- 자동 스케줄링 (향후 추가 예정)
- 로그인이 필요한 게시판 크롤링
- 이미지/첨부파일 수집 (텍스트만)
- 실시간 알림

## Style
- 기존 docstore UI와 동일한 디자인 시스템 (다크모드, 카드 기반)
- 업로드 탭 내에 크롤링 섹션 추가 또는 별도 탭

## Key Concepts
- **지식화**: 크롤링된 외부 콘텐츠를 docstore 문서로 변환 (섹션 분리 + 벡터 임베딩)하여 RAG 검색 대상에 포함시키는 것
- **점수 시스템**: 제목/내용 키워드 매칭에 차등 가중치를 적용하여 관련성 높은 결과를 상위에 정렬
- **크롤링 소스**: 게시판 URL + 게시판 구조 정보(CSS selector 등)를 등록하여 반복 수집 가능하게 하는 설정 단위

## Open Questions
- 게시판 HTML 구조가 변경되면 selector 업데이트 필요 — 관리 방안은 향후 결정
