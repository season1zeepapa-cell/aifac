# Brief: 바이브코딩 기반 법률·보안 검토 도구

> 채팅/파일 입력 내용을 법률과 정보보호 기준으로 자동 식별하고, 근거 조항까지 제시하는 내부용 검토 솔루션.

## Requirements
- [ ] 1. 사용자는 채팅 텍스트와 파일(`pdf/docx/txt/md`)을 업로드해 단일 요청으로 분석할 수 있다.
- [ ] 2. 5,000자 텍스트 기준 분석 결과가 10초 이내에 반환된다(초기 목표).
- [ ] 3. 결과는 `법률`/`보안` 2개 분류로 나뉘고, 각 분류별 `요약`과 `상세내역` 2단계로 제공된다.
- [ ] 4. 각 검토 항목은 근거를 포함한다: 관련 문장, 법령/규정명, 조항(또는 항목) 번호, 검토 필요 사유.
- [ ] 5. 법률 기준은 대한민국 법제처 법령정보를 기반으로 하며, `자동 동기화`와 `수동 목록 관리`를 모두 지원한다.
- [ ] 6. 보안 기준은 사내 정보보호 규정 + 추가 기준으로 관리되며, 등록/수정/삭제는 관리자만 가능하다.
- [ ] 7. 추가 기준 데이터는 암호화 저장하고, 조회/변경 이력(누가/언제/무엇)을 감사로그로 남긴다.
- [ ] 8. 기준(법률/보안) 확장 시 기존 분석 기능이 깨지지 않도록 버전 관리와 롤백 절차를 제공한다.

## Constraints
- 대한민국 법령 체계와 한국어 문서 검토를 1차 범위로 한다.
- 내부용 도구로 운영하며, 권한 없는 사용자에게 기준 원문/세부 규칙을 노출하지 않는다.
- 관리자 기능은 인증된 관리자 계정에서만 접근 가능해야 한다.

## Non-goals
- 법률 자문 또는 최종 법적 판단을 자동으로 대체하지 않는다.
- 결과만으로 자동 승인/반려를 수행하지 않는다(사람 검토 필수).
- 초기 버전에서 다국가 법령 동시 지원은 포함하지 않는다.

## Style
- 실무자가 바로 판단할 수 있게 간결하고 근거 중심으로 보여준다.
- 경고 위주 표현보다 “왜 검토가 필요한지”를 설명하는 검토 보조형 UX를 사용한다.
- 요약은 빠른 의사결정용, 상세는 감사/추적 가능한 증적용으로 분리한다.

## Key Concepts
- **바이브코딩**: 빠른 반복으로 필요한 내부 도구를 점진적으로 완성하는 개발 방식.
- **검토 필요 포인트**: 법률/보안 기준에 의해 추가 확인이 필요한 문장 또는 파일 구간.
- **법률 기준 엔진**: 법제처 법령정보를 동기화/관리해 법적 검토 포인트를 찾는 기준 체계.
- **보안 기준 엔진**: 사내 규정과 추가 기준으로 개인정보/정보보호 리스크를 식별하는 체계.
- **2단계 결과**: 1차 요약(핵심 위험), 2차 상세(근거 조항·사유·원문 위치) 제공 방식.

## Open Questions
- 법제처 동기화 주기(실시간/일 1회/주 1회)와 장애 시 fallback 정책 확정.
- 업로드 파일 최대 용량과 허용 확장자 세부 정책 확정.
